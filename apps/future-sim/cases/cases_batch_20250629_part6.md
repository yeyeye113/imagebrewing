## 一、短脉冲衰减案例（Short Burst Decay）

### Case 1: Yo App — 通知即产品的极端案例

| 维度 | 详情 |
|------|------|
| 名称 | Yo |
| 类型 | 极简通知App（只能发送"Yo"一个词） |
| 时间线 | 2014年4月发布 → 数周内达2-3M下载 → 2014年底淡出主流视野 |
| 关键指标 | 峰值2-3M周下载；App Store 31国第一；融资150万美元；下载量下降90%+仅在数周内 |
| 衰减曲线 | 天级别衰减 — 上线即巅峰，数周内跌回接近零 |
| 核心机制 | **纯新奇驱动** — 产品本身无任何实用价值，唯一的传播动力是"这太荒谬了"的社交话题性 |
| 渠道依赖 | Twitter/social media话题传播 |
| 为什么失败 | 零留存钩子（无内容、无feed、无每日打开理由）；零网络效应差异（给1人发Yo和给100人发Yo体验完全一样）；Apple因安全问题一度下架 |
| 衰减类型 | 新奇燃尽型（Novelty Burn） |

### Case 2: Flappy Bird — 开发者亲手终结的病毒

| 维度 | 详情 |
|------|------|
| 名称 | Flappy Bird |
| 类型 | 2D像素风飞行躲避游戏 |
| 时间线 | 2013年5月发布 → 2014年1月因PewDiePie视频爆发 → 2014年2月10日开发者主动下架 |
| 关键指标 | 峰值日收入5万美元（广告）；App Store第一；手机预装版eBay拍价超9万美元；巅峰期每天60+仿品上线 |
| 衰减曲线 | 非自然衰减 — 不是用户流失，而是开发者Dong Nguyen主动从App Store和Google Play下架 |
| 核心机制 | **挫败感传播** — 人们下载不是为了玩，是为了理解"这游戏为什么这么火"，形成病毒传播 |
| 渠道依赖 | YouTube（PewDiePie评测是引爆点）；社交媒体话题传播 |
| 为什么消失 | 开发者声称游戏"毁了他的正常生活"，主动下架；产品无进度系统、无社交功能、无留存机制；Metacritic评分52/100 |
| 衰减类型 | 主动终止型（Developer-Initiated Shutdown） |

### Case 3: HQ Trivia — 直播答题的99.4%坠落

| 维度 | 详情 |
|------|------|
| 名称 | HQ Trivia |
| 类型 | 实时直播答题游戏App |
| 时间线 | 2017年8月发布 → 2018年3月28日巅峰（238万同时在线）→ 玩家降至约1.5万 → 2020年2月停运 → 2022年11月最后一场游戏 |
| 关键指标 | 峰值238万同时在线；后期仅1.5万玩家/场；衰减幅度99.4%；融资1500万美元（Founders Fund）；估值目标曾达1亿 |
| 衰减曲线 | 12个月内从238万降至1.5万 — 断崖式衰减 |
| 核心机制 | **时间窗口摩擦** — 必须在特定时间在线参与，错过即无法参与；技术不稳定（卡顿、误判淘汰）；奖金池缩水至1500美元失去吸引力 |
| 渠道依赖 | Twitter话题传播 + 媒体报道"百万同时在线"的奇观效应 |
| 为什么失败 | 异步参与摩擦（必须定时上线）；技术问题频发；奖金缩水失去参与动力；联合创始人Colin Kroll 2018年12月去世；商业化失败（23个节目中仅3个获得赞助） |
| 衰减类型 | 同步参与摩擦型（Scheduling Tax） |

### Case 4: BeReal — 每日通知的69%坠落

| 维度 | 详情 |
|------|------|
| 名称 | BeReal |
| 类型 | 每日定时拍照分享App（前后摄像头同时拍摄） |
| 时间线 | 2020年发布 → 2022年7月登顶App Store → 2022年8月峰值7350万月活 → 2024年初降至约2300万月活 → 2024年6月被Voodoo以约5亿欧元收购 |
| 关键指标 | 峰值7350万MAU；DAU从1500万降至约600万（7个月内降60%）；2024年巴黎奥运会期间仅新增1000用户；融资8500万（估值6亿） |
| 衰减曲线 | 18个月内从7350万降至2300万 — 稳定持续衰减 |

| 衰减类型 | 新奇燃尽型（Novelty Burn） |

### Case 5: Draw Something — Zynga在巅峰日收购的悲剧

| 维度 | 详情 |
|------|------|
| 名称 | Draw Something |
| 类型 | Pictionary式手机绘画猜词游戏 |
| 时间线 | 2012年2月6日发布 → 2012年3月21日峰值1500万DAU（Zynga当天宣布收购）→ 2012年5月降至1000万DAU → 2012年12月关闭 |
| 关键指标 | 5周内2000万下载；峰值1500万DAU；Zynga以1.8亿美元收购（恰好在峰值日）；5周内DAU下降33%（500万用户流失） |
| 衰减曲线 | 5周内从1500万降至1000万 — 收购即巅峰 |
| 核心机制 | **社交分享新奇消退** — 人们分享绘画给朋友看的社交动力消退后，单人游戏循环缺乏深度 |
| 渠道依赖 | 社交媒体分享传播（用户把画作分享到Facebook/Twitter引发下载） |
| 为什么失败 | 病毒性传播在社交网络上的新鲜感消退；无进度系统留存玩家；单局游戏循环重复；Zynga的变现重心可能赶走了休闲玩家 |
| 衰减类型 | 收购即巅峰型（Acquisition-at-Peak Trap） |

### Case 6: Threads by Meta — 100M用户5天后的80%坠落

| 维度 | 详情 |
|------|------|
| 名称 | Threads |
| 类型 | 文字社交网络App（Instagram生态，Twitter竞品） |
| 时间线 | 2023年7月5日发布 → 7月7日峰值4930万DAU（Android）→ 7月17日DAU降70% → 8月DAU降80%+ → 后续逐步恢复至2025年4亿MAU |
| 关键指标 | 2小时200万用户；5天1亿用户（史上最快增长消费级应用）；6周内DAU降80%+；用户停留时间从20分钟降至不足3分钟 |
| 衰减曲线 | 6周内从4930万降至约1030万（Android）— 爆发即巅峰 |
| 核心机制 | **Instagram导流 + 功能缺失** — 用户因Instagram一键注册而来，但产品缺乏搜索、趋势话题、关注列表等关键功能；算法推荐feed让用户无法自主控制 |
| 渠道依赖 | Instagram账号绑定（删除Threads需删除Instagram）；纯平台导流 |
| 为什么失败 | 缺少关键功能（搜索、趋势、关注feed）；无创作者/KOL生态；Instagram绑定带来注册但非真实需求；Twitter/X保留了其核心用户和内容 |
| 衰减类型 | 平台导流虚假繁荣型（Platform-Inflated Vanity） |

### Case 7: Vine — 6秒视频的创作者大逃亡

| 维度 | 详情 |
|------|------|
| 名称 | Vine |
| 类型 | 6秒循环短视频平台 |
| 时间线 | 2012年10月Twitter收购 → 2013年1月iOS发布 → 2013年4月App Store第一 → 2015年12月2亿+活跃用户 → 2016年1-6月超半数头部创作者停更 → 2017年1月关闭 |
| 关键指标 | 峰值2亿+活跃用户；6个月内50%+头部创作者（15K+粉丝）停更或删除账号；Twitter以约3000万收购 |
| 衰减曲线 | 6个月内头部创作者流失超50% — 创作者先于用户离开 |
| 核心机制 | **创作者变现真空** — 无收入分成机制，头部创作者发现YouTube/Instagram能赚钱后集体迁移 |
| 渠道依赖 | Twitter生态整合是增长引擎；当Twitter停止优先级支持后，分发渠道崩塌 |
| 为什么失败 | Twitter未投入创作者变现工具；Instagram推出竞品功能；6秒限制对严肃创作者是创作枷锁；营销人员转向能讲完整故事的长视频平台 |
| 衰减类型 | 创作者出逃型（Creator Exodus） |

### Case 8: Yik Yak — 校园社交的76%年衰减

| 维度 | 详情 |
|------|------|
| 名称 | Yik Yak |
| 类型 | 匿名地理位置社交App（校园场景） |
| 时间线 | 2013年11月发布 → 2014年估值3.5-4亿（Sequoia领投6000万+）→ 2016年下载量降76% → 2017年4月宣布关闭 → Square以约100万收购残余资产 |
| 关键指标 | 峰值估值4亿；年下载量下降76%；资产最终以100万卖出（99.7%损失）；50人裁员30人 |
| 衰减曲线 | 12个月内从巅峰走向关闭 — 校园场景的生命周期极短 |
| 核心机制 | **场景依赖 + 内容毒性** — 匿名+地理位置在校园场景有天然传播力，但导致大规模网络霸凌（种族主义、反犹太主义）；多州学校封禁；被曝光操纵竞品帖子投票 |
| 渠道依赖 | 校园口碑传播 + 匿名社交的新奇感 |
| 为什么失败 | 网络霸凌导致学校封禁；无商业模式（即使用户大量流失也无收入）；匿名+地理位置是隐私隐患；校园用户群会自然"老化"，新一届学生不会继承App |
| 衰减类型 | 场景依赖+毒性死亡型（Toxic Community Collapse） |

### Case 10: Juicero — 硅谷媒体泡沫的破灭

| 维度 | 详情 |
|------|------|
| 名称 | Juicero |
| 类型 | $699（后降至$399）Wi-Fi联网榨汁机，使用专有蔬菜包 |
| 时间线 | 2016年3月发布（$699）→ 2017年1月降价至$399 → 2017年9月1日停止运营 |
| 关键指标 | 累计融资1.2亿美元；232名员工；从未实现有意义的商业规模 |
| 衰减曲线 | 18个月内从"硅谷宠儿"到关门 — 一个Bloomberg视频摧毁了整个价值主张 |

| 渠道依赖 | 硅谷科技媒体炒作 + Product Hunt式产品曝光 — 媒体热度就是产品本身 |
| 为什么失败 | 核心价值主张（必须用机器才能榨汁）被手动挤压演示完全摧毁；蔬菜包5-7美元/包且保质期仅8天；$699硬件+持续购买包的商业模式在价值被否定后无法成立 |
| 衰减类型 | 叙事破灭型（Narrative Collapse） |

### Case 11: Clubhouse — 邀请制FOMO的21天21%衰减

| 维度 | 详情 |
|------|------|
| 名称 | Clubhouse |
| 类型 | 实时音频社交App |
| 时间线 | 2020年3月发布 → 2021年1月200万WAU → 2021年2月1000万WAU（巅峰）→ 3周内降21% → 2023年4月裁员50% |
| 关键指标 | 邀请码eBay售价$400；3周内WAU降21%（约1%/天）；Android版2021年5月才上线（错过巅峰窗口）；估值40亿 |
| 衰减曲线 | 巅峰后3周内降21% — 开放即衰退 |

| 为什么失败 | 开放邀请制后排他性消失；Twitter Spaces零成本复制核心功能；同步参与需要用户在特定时间在线；1.3M用户数据泄露（2021年4月）侵蚀信任 |
| 衰减类型 | 稀缺性崩塌型（Scarcity Illusion Collapse） |

## 二、渠道依赖案例（Channel-Dependent）

### Case 12: Meerkat — Twitter API断供即死

| 维度 | 详情 |
|------|------|
| 名称 | Meerkat |
| 类型 | 移动端直播App |
| 时间线 | 2015年3月SXSW爆发 → 2015年3月Twitter切断API → Twitter推出Periscope → Meerkat迅速死亡 |
| 关键指标 | SXSW期间爆发式增长；融资1200万（估值5000万）；Twitter切断API后数月内基本消亡 |
| 核心依赖 | **Twitter社交图谱API** — 用Twitter账号导入好友关系，用Twitter Feed分发直播 |
| 为什么危险 | 增长完全建立在竞争对手的API上；Twitter可以随时切断（事实也确实这么做了）；当Twitter收购Periscope并推出自家竞品时，Meerkat的分发渠道瞬间归零 |
| 教训 | **永远不要在竞争对手的平台上建立分发** — 他们可以用一个API策略变更杀死你 |

### Case 15: Parler — 建立在"敌人"基础设施上的愤怒经济

| 维度 | 详情 |
|------|------|
| 名称 | Parler |
| 类型 | "言论自由"社交网络 |

### Case 16: Gab — 100万注册 vs 1.9万活跃的鸿沟

| 维度 | 详情 |
|------|------|
| 名称 | Gab |
| 类型 | 另类科技微博客 |
| 时间线 | 2016年发布 → 注册用户约100万 → 活跃用户约1.9万 → 持续依赖主流平台封禁事件获取用户 |

### Case 17: Google+ — 强制捆绑制造的虚假繁荣

| 维度 | 详情 |
|------|------|
| 名称 | Google+ |
| 类型 | Google的社交层 |
| 时间线 | 2011年6月发布 → 5.4亿"月活"（含强制绑定）→ 90%的session持续不到5秒 → 2018年关闭消费者版 → 2019年完全关闭 |
| 核心依赖 | **Google产品线强制捆绑** — YouTube评论、Gmail、Android都强制绑定Google+ |
| 为什么危险 | 5.4亿"月活"是虚数 — 用户被迫注册而非主动选择；90%的session <5秒说明用户进来就走；强制分发制造的是怨恨而非采用 |
| 教训 | **强制分发可以制造注册数据，但无法制造产品市场契合** — 指标可以完全掩盖PMF缺失 |

### Case 18: Myspace — Google广告协议的慢性自杀

| 维度 | 详情 |
|------|------|
| 名称 | Myspace |
| 类型 | 社交网络（早期霸主） |
| 时间线 | 2003年发布 → 2006年成为美国访问量最大网站（1.15亿月访客）→ 2008年与Google签9亿广告协议 → 广告体验恶化 → 2011年以3500万卖给Specific Media |
| 核心依赖 | **Google广告协议（$900M担保）+ 音乐产业** — 广告收入是核心商业模式 |
| 为什么危险 | Google协议要求激进的广告投放，严重损害用户体验；当Facebook提供更干净的替代品时，用户迅速迁移；广告优先的策略导致产品路线图被广告需求绑架 |
| 教训 | **单一广告合作伙伴依赖可以绑架产品路线图** — 为了满足$900M协议而牺牲用户体验，最终两个都失去了 |

### Case 19: HQ Trivia — Twitter话题依赖

| 维度 | 详情 |
|------|------|
| 名称 | HQ Trivia |

## 三、衰减模式分类总结

### 模式 A：新奇燃尽型（Novelty Burn）
**特征**: 产品核心机制有趣一周，然后变得重复。无深度、无进度、无留存理由。
**案例**: Yo App, BeReal, Draw Something
**关键指标**: 首周留存率极低；DAU/MAU比值快速下降

### 模式 B：FOMO崩塌型（Scarcity Illusion Collapse）
**特征**: 人工稀缺/排他性制造巨大FOMO，开放后价值主张蒸发。
**案例**: Clubhouse, Lemon8
**关键指标**: 开放/开放后WAU立即下降；邀请码价格归零

### 模式 C：平台导流虚假繁荣型（Platform-Inflated Vanity）
**特征**: 增长来自平台导流（Instagram绑定、强制捆绑），非真实产品需求。
**案例**: Threads, Google+
**关键指标**: 注册数高但session时长极短；DAU/注册比极低

### 模式 D：创作者出逃型（Creator Exodus）
**特征**: 平台依赖创作者产生内容，但无变现机制导致创作者迁移。
**案例**: Vine, Yik Yak
**关键指标**: 头部创作者活跃度骤降；内容产出量下降

### 模式 E：渠道断供型（Channel Cut-Off）
**特征**: 增长完全依赖一个渠道（API、分发、推荐），渠道断供即死。
**案例**: Meerkat, Periscope, Gab, Parler
**关键指标**: 渠道变更后用户获取量断崖；注册量与活跃量严重脱节

### 模式 F：叙事破灭型（Narrative Collapse）
**特征**: 产品价值主张被一次简单演示/事件彻底摧毁。
**案例**: Juicero, Flappy Bird
**关键指标**: 媒体热度从正面转为负面的转折点；搜索量从飙升转为暴跌

### 模式 G：同步参与摩擦型（Scheduling Tax）
**特征**: 要求用户在特定时间参与，异步参与不可能。
**案例**: HQ Trivia, Clubhouse
**关键指标**: 参与率随时间推移持续下降；回访用户比例极低

### 模式 H：场景依赖+毒性死亡型（Toxic Community Collapse）
**特征**: 产品在特定场景（校园）有传播力，但内容毒性导致封禁和用户逃离。
**案例**: Yik Yak
**关键指标**: 平台封禁事件数；负面媒体报道量

## 五、关键教训

1. **分布渠道是租来的，不是拥有的** — 唯一可防御的护城河是用户主动寻找并回访的产品，无论他们最初在哪里发现它
2. **注册数≠用户数** — Gab有100万注册但仅1.9万活跃；Google+有5.4亿"月活"但90%的session不到5秒
3. **奇观效应是一次性的** — 百万人同时答题的奇观无法重复制造
4. **FOMO是营销策略，不是留存策略** — 邀请制创造了排他感，但开放后价值蒸发
5. **创作者需要变现才能留下** — Vine的6秒限制是创意枷锁，但真正的死因是零收入分成
6. **平台子产品的生死取决于母体决策** — Periscope、Vine、Lemon8都因母体/平台的战略变化而终结
7. **单一演示可以摧毁整个价值主张** — Juicero的手动挤压视频证明了$699机器的荒谬性
8. **愤怒驱动的用户零忠诚度** — Parler和Gab的"难民"用户随时会离开
9. **同步参与摩擦是致命的** — 要求用户在特定时间上线的产品注定衰减
10. **场景依赖产品有自然生命周期** — 校园社交的用户群会"老化"，新一届学生不会继承App
